# 快速入门
声明：本项目的核心底层交互基于优秀的开源框架 mineflayer 开发，特此致谢！本项目主要致力于提供开箱即用的可视化 UI 界面，降低新手使用门槛。
欢迎使用 ARC's Agent！本指南将手把手教你如何通过控制面板快速配置并召唤你的专属 AI 机器人。

## 第一步：环境准备
在开始创建机器人之前，请先进入**设置**页面完成基础环境的配置：
1. 下载并安装 **Node.js 运行时**。
2. 下载机器人运行所需的**基础包**。

## 第二步：创建机器人
环境准备就绪后，回到首页点击**创建机器人**。接下来请根据提示填写各项配置：

### 1. 服务器配置
* **服务器地址 (MC_HOST)**：填写你要加入的服务器 IP（如果是你本地开的服务器，请填写 `localhost`）。
* **端口 (MC_PORT)**：服务器端口（默认通常为 `25565`）。
* **版本 (MC_VERSION)**：填写游戏版本。如果是代理/群组服务器，建议使用 `1.20.1` 并配合 ViaVersion 插件跨版本。
* **用户名 (MC_USERNAME)**：给你的机器人起个名字（即它在游戏内的 ID）。
* **认证方式 (MC_AUTH_TYPE)**：正版账号选择 `microsoft`，离线服务器选择 `offline`。
* **登录密码 (MC_LOGIN_PASSWORD)**：选填。如果服务器有登录插件（如 AuthMe），填入密码后机器人会自动执行 `/l 密码` 进行登录。如果服务器没有密码请留空。
* **主人名称 (MC_OWNER_NAME)**：填写**你自己的游戏 ID**。机器人非常忠诚，它只会听从主人的指令。

### 2. AI 配置
* **API Key (LLM_API_KEY) & API URL**：这是机器人的“大脑”，需要你自己购买并填写。
  * *提示：如果不了解如何获取，可以询问 AI。推荐使用 DeepSeek，价格比较便宜，性价比极高。如果暂时无法购买，也可以去寻找提供免费体验额度的 API。*
* **对话模式**：
  * 默认推荐使用 **whisper（私聊模式）**，这需要服务器安装支持 `/tell` 指令的插件。
  * 如果服务器没有安装相关插件，请选择 **public（公屏模式）**。

### 3. 行为设置
在行为设置中，你可以开启机器人的各种“被动本能”：
* **登录自动传送 (INSTINCT_AUTO_TP_LOGIN)**：**强烈建议勾选**！开启后，机器人会自动接受主人的 `/tpahere` 传送请求，省时省力。
* 你还可以根据需求勾选自动防御、自动进食、自动切换工具、自动丢弃垃圾等实用功能。

---

## 第三步：启动与交互

保存配置并创建后，启动机器人，它应该就能成功加入游戏了！
> ⚠️ **注意**：机器人加入游戏时可能会触发某些服务器的反作弊系统。如果被踢出，请检查服务器的反作弊设置或调整机器人的行为频率。

### 如何给机器人下达指令？
* **游戏内指令**：如果使用的是 `whisper` 模式，请在游戏聊天框输入：
  ```text
  /tell <机器人名字> <你想让它做的事>
  ```
  *例如：`/tell MyBot 过来一下` 或 `/tell MyBot 帮我挖32个铁矿石`*
  （注意：机器人只听主人的话！）
* **控制台交互**：你在控制台输入的所有内容，都会被机器人直接发送到游戏中。

## 💡 机器人能做什么？（指令参考）
你的机器人接入了 AI 大脑，你可以直接用自然语言（大白话）给它下达指令！以下是一些常用的指令示例：

* **移动与跟随**：过来一下 / 跟着我 / 去坐标 100 64 -200 那里等我
* **采集与挖掘**：帮我挖32个铁矿石 / 去附近砍一组橡木原木 / 去附近帮我剪点羊毛
* **加工与合成**：帮我合成一个工作台并放下 / 帮我把身上的粗铁烧成32个铁锭
* **组合任务**：帮我挖3个铁矿，找点煤炭，然后烧成3个铁锭
* **建筑与破坏**：在你脚下放一个箱子 / 帮我用橡木木板把坐标 0 64 0 到 10 64 10 这片区域填平
* **农业与养殖**：帮我把这些种子种下去 / 用骨粉把附近的庄稼催熟一下 / 拿点小麦去喂一下附近的牛
* **战斗与安保**：去帮我杀几只僵尸 / 去揍那个叫 Steve 的玩家 / 在附近30格范围内巡逻一下
* **物品管理**：把你身上的东西都放进箱子里 / 从箱子里帮我拿一组钻石出来 / 扔给我一把剑

## 🛠️ 故障排除
在复杂的 Minecraft 世界中，机器人偶尔也会遇到一些小麻烦，你可以用以下指令控制它：
* **卡住不动了？** 机器人底层有“自动脱困”机制，通常稍微等待一下即可。
* **任务执行失败 / 卡循环？** 
  * 发送 `/tell <机器人名字> retry`：让机器人重新尝试当前任务。
  * 发送 `/tell <机器人名字> skip`：跳过当前任务，执行下一步。
* **想让它立刻停下？** 
  * 发送 `/tell <机器人名字> stop`：机器人会紧急刹车并清空当前的任务计划。
* **遇到无法解决的 Bug？**
  * 直接在面板关闭机器人然后重新开启即可，重启能解决 99% 的问题！
